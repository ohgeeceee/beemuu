      Session & security
      Locked

      Security unlock uses the module's seed/key algorithm. Only the simulator's algorithm is built in; real BMW modules need their per-ECU algorithm added in commands.rs. Some service functions require an unlocked extended session.

      Service functions

      Entries tagged UNVERIFIED use routine IDs that have not been validated on a real chassis (simulator-grade). Running one shows an extra confirmation. Validation harness: docs/validation/service-functions.md.
