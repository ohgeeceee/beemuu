## Problem

The repo's user-facing docs contradict the actual shipped state:

| Item in docs | Label in docs | Actual status (verified against `origin/main`) |
|---|---|---|
| Hunt tab (README § Features table) | listed as current feature | ❌ **Removed in commit `43b7171` (PR #57)** — `hunt.rs/hunt.js/hunt.css` and scoring hooks deleted from the desktop app. |
| Diagnostic Story (README § "What's coming") | "in progress" | ✅ **Shipped in v0.3.0** — `src-tauri/src/story.rs`. |
| Community Oracle (README § "What's coming") | "research" | ✅ **Shipped in v0.3.0** — `src-tauri/src/oracle.rs`. |
| All six decode functions (ROADMAP § v0.3.0) | "🟢 Ready, blocks ~40% of data" | ✅ **Shipped** — live in `src-tauri/src/data/live.rs` with unit tests; corresponding DIDs uncommented in `community/profiles/b58.toml` and `community/profiles/n55.toml`. |
| Adaptive Drift Tracker | unlabeled, no roadmap entry | ⚠️ Aspirational; only in README + `LAUNCH_POST.md`. |
| Tuning Fingerprint Detector | "research", no roadmap entry | ⚠️ Aspirational; only in README. |

A user reading the README + roadmap today cannot tell which bullets are
shipped, which are scheduled, and which are aspirational.

## Fix

1. README § "What's coming" → split into three subsections: **v0.4.0 Ready
   items**, **Recently shipped (v0.3.0)** with file links, and **Ideas
   being explored (not on the roadmap)**.
2. ROADMAP § v0.3.0 → flip decode-fn rows to ✅ Done; mark `u8_enum` as
   🟡 Deferred (the one that genuinely didn't ship).
3. ROADMAP § v0.4.0 → rewrite with explicit Ready / Needs-research /
   Deferred-to-v0.5 splits.
4. CHANGELOG `[Unreleased]` → one Added bullet (roadmap + this spec) and
   one Changed bullet (README drift fix).

## Why this is the first PR for v0.4

- Smallest possible change that improves user-facing truth.
- Doc-only, so it qualifies for auto-merge per `CLAUDE.md` rule 2
  ("only doc-only PRs may auto-merge").

  (Histograms, CBS reset, `u8_enum` decoder, etc.) is the first
  *code* PR for v0.4.
