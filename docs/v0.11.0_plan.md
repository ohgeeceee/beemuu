## Premise

Owners run a logging session, see a knock sensor spike at 4200 rpm,
and want to ask someone who knows — a forum thread, a friend, a
subreddit. The **first thing** they need is the picture: a chart they
can paste straight into a post, and an opinion library they can skim
without running the app. BeeEmUu already logs, draws, histograms,
plays back, and merges traces; the *export* surface is the gap. v0.9.0
and v0.10.0 closed the diagnostic-workflow gaps (guided plans,
verification badges). v0.11.0 closes the **share-the-trace** gap —
the cycle an owner actually finishes a session with.

PNG export is the first half. The second half is the format the
diagnostic community already standardises on (CSV), the second
*visual* (SVG for crisp forum embedding), and the surface that makes
a shared log reviewable without an ENET cable: a static, self-contained
HTML walkthrough that replays the trace in any browser.

The honest ceiling is the same one every prior cycle shipped under:
community-sourced formats, no proprietary ISTA+ export shape, no
upload to third-party telemetry services, no write paths through
this surface (export is read-only by construction).

## Survey — the existing trace surface

| Subsystem | What it produces today | Where it lives | What an owner can do with it |
|---|---|---|---|
| Logging | ~4 Hz CSV in `~/Documents/beemuu/logs/` | `src-tauri/src/data/logging.rs` | Open in Excel / share file |
| Chart rendering | Chart.js lines (logging panel) + histograms | `src/js/chart.js`, `src/js/histogram.js` | Look at it in-app; nothing else |
| Trace comparison | Diff modal merges two logs side-by-side | `src/js/merge.js` | Same as above |
| Walkthrough | Step-by-step panel in fault detail | `src/js/walkthrough.js` (v0.9.0 PR #4) | App-only |
| Freeze frames | Per-ECU schema decode | `src-tauri/src/data/freeze.rs` | In-app detail panel |

What it **cannot** do: export the chart visually; export the CSV in a
forum-friendly wrapper (timestamp column header, units in row 2);
embed the walkthrough in a static share; produce an SVG that survives
a 4K forum upload; bundle a session for offline review on a phone.

## What we will NOT do (this cycle)

- **Cloud upload / telemetry.** No `POST` to a third-party host. Export is local-only; if a user wants to share it, they choose where.
- **ISTA+ / proprietary BMW export shapes.** Copyright + reverse-engineering ceiling; same reason v0.8.0 drew the line on programming and FSC.
- **PDF report.** Layout engine is a quarter-cycle of work; revisit after the smaller formats prove out.
- **Multi-language export headers (DE/EN/FR/CN).** Lifted from the backlog; needs a localisation pass across the logging surface first, not just the export.
- **Walkthrough auto-share.** The static HTML bundle is opt-in (button click → file save). No auto-upload, no "share to…" sheet.

## Why "Share the Trace" as the cycle name

Cycle names track the user promise. v0.8.0 made BeeEmUu the service
bay; v0.9.0 made it the diagnostician; v0.10.0 made its advice
honest about what was verified. v0.11.0 makes the trace something
the owner can actually hand to someone else — the cycle that ends
the session. The name is deliberately the verb an owner does
after they're done diagnosing, not the tool's internal vocabulary.
